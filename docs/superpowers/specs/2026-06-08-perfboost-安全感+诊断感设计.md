# PerfBoost 安全感+诊断感 改进设计

> 日期: 2026-06-08 | 状态: 已确认

## 定位

> **比 CCleaner 干净，没广告没捆绑，删得还更安全**

核心用户：普通家庭用户（电脑小白），帮自己 + 帮身边朋友家人解决电脑卡顿。

## 改进范围

| 优先级 | 模块 | 做什么 | 解决什么痛点 |
|--------|------|--------|-------------|
| **P0** | 垃圾清理 | 安全三级分级 🟢🟡🔴、默认只勾绿色、大白话解释、标注清理范围 | "怕删错东西" |
| **P0** | 仪表盘 | 一句话诊断，区分 C 盘（可清理）和 D/E 盘（提示手动整理） | "看不懂数字" |
| **P1** | 启动项 | 增加开机耗时预估 | "关了有什么好处" |
| **P2** | 进程管理 | 维持现状、小修 | — |

---

## 1. 垃圾清理 — 安全分级

### 1.1 当前问题

- 7 个清理类别平铺展示，小白不知道哪些安全哪些危险
- 缺少大白话解释，用户不敢勾选
- 未标注"清理范围主要是 C 盘"，用户可能误以为能清理所有磁盘

### 1.2 改进设计

#### 风险三级分级

| 级别 | 标签 | 含义 | 默认状态 |
|------|------|------|---------|
| 🟢 安全 | 绿色 | 删了完全不影响系统和软件正常使用 | ✅ 全部勾选 |
| 🟡 谨慎 | 黄色 | 需要管理员权限，或删除后需要重新生成（如缩略图） | ❌ 不勾选 |
| 🔴 高级 | 红色 | 可能影响问题排查，仅建议高级用户操作 | ❌ 不勾选 |

#### 类别分配

| 类别 | 风险 | 大白话解释 |
|------|------|-----------|
| 用户临时文件 | 🟢 安全 | 软件运行时产生的临时文件，删了完全不影响使用 |
| 浏览器缓存 | 🟢 安全 | 浏览器存的网页图片和文件，删了不影响密码和收藏，网页重新加载即可 |
| 回收站 | 🟢 安全 | 你已经删过的文件，清空后释放它们占用的空间 |
| 错误报告 | 🟢 安全 | 软件崩溃时生成的调试文件，对普通用户没任何用处 |
| 系统临时文件 | 🟡 谨慎 | Windows 自己的临时文件，一般安全但需要管理员权限 |
| 缩略图缓存 | 🟡 谨慎 | 文件夹里图片视频的预览小图，删了下次打开文件夹会重新生成，略慢 |
| Windows 日志 | 🔴 高级 | 系统运行记录，出问题时排查用，建议保留 |

#### 页面顶部提示

> 💡 清理范围：系统临时文件、浏览器缓存等，主要释放 **C 盘**空间

#### 交互要点

- 全选按钮 **只切换 🟢 类别**，不碰 🟡🔴
- 🟡🔴 类别需用户手动勾选
- 每个类别卡片用对应颜色边框/标记区分

## 2. 仪表盘 — 一句话诊断

### 2.1 当前问题

- 展示 CPU、内存、磁盘、网络四张数据卡片，用户看不懂数字意味着什么
- 所有磁盘等同展示，未区分 C 盘（可清理）和其他盘（无法清理）

### 2.2 改进设计

#### 一句话诊断

四张卡片上方显示自动诊断结论：

```
状态规则：
├─ C 盘 < 10GB 剩余 → 🔴 C 盘剩余 X GB，建议清理垃圾释放空间
├─ 内存 > 85%     → 🔴 内存占用较高，可以结束一些后台应用
├─ CPU 持续 > 80%  → 🟡 CPU 负载较高，可在进程页查看原因
├─ 其他盘 < 10GB   → 🟡 D/E 盘剩余 X GB，建议手动整理文件
└─ 全部正常        → 🟢 你的电脑运行良好
```

#### 磁盘卡片区分

| 盘符 | 紧张时处理 |
|------|-----------|
| **C 盘** | 提示"可清理系统垃圾释放空间"，引导用户去清理页 |
| **D/E 等其他盘** | 提示"空间不足，建议手动整理文件"，不给操作入口（垃圾清理管不到这些盘） |

## 3. 启动项管理 — 看得懂的决策信息

### 3.1 当前问题

- 显示"程序文件存在/不存在"，对小白无意义
- 用户不知道关了启动项具体有什么好处

### 3.2 改进设计

每个启动项卡片新增一行：

> ⏱ 预估拖慢开机 **X 秒**

#### 估算逻辑（粗略方案）

- 根据程序文件大小、所在磁盘类型（SSD/HDD）给出粗略估值
- 核心目的不是精确计时，而是给用户一个"关掉的理由"
- 对文件不存在的启动项标注"该程序已卸载，可安全禁用"

---

## 4. 进程管理 — 维持现状

进程管理对小白用户不是刚需功能，当前版本已具备基本能力（查看、搜索、结束进程、保护系统进程）。

仅做维护性小修，不投入大的重构。

---

## 非目标（本次不做）

以下功能明确延后，保持聚焦：

- 一键优化首页（与现有清理功能无本质区别）
- 大文件/重复文件扫描（D/E 盘空间管理方案，P2）
- 注册表清理（P2）
- 隐私清理（P2）
- 定时自动清理（P2）
- 系统还原点创建（增加复杂度但非核心便捷性）
- 撤销中心/已清理记录恢复（增加复杂度但非核心便捷性）

---

## 设计原则

1. **帮用户判断，而不是让用户判断** — 筛好安全项，默认勾好，用户只管确认
2. **说人话，不说术语** — 每处文字都以"我妈能看懂"为标准
3. **明确告知边界** — 做的事说清楚，不做的事也说清楚，管理预期
4. **轻量克制** — 不加功能和按钮除非它解决一个明确的用户问题
